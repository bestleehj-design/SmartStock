# 策略2实现说明

## 概述

策略2是从策略1筛选出的股票中，进一步筛选出热点板块的股票。这符合需求文档中"从策略1中选出第二天就要买的票"的要求。

## 实现逻辑

### 第一层：策略1筛选
1. **技术条件**：
   - 5/10/30日线粘合
   - 量价齐升（放量 ≥ 1.5倍，涨幅 ≥ 3%）
   
2. **过滤条件**：
   - 必须是上涨的股票（`change_pct > 0`）
   - 下跌的股票会被排除

### 第二层：热点板块筛选
根据需求文档"策略2-方法1：跟随热点板块"：

1. **识别热点板块**：
   - 统计策略1候选股票中各行业（`industry`）的出现次数
   - 将出现2只及以上股票的行业识别为热点板块

2. **筛选逻辑**：
   - 从策略1结果中筛选出属于热点板块的股票
   - 如果未识别到热点板块，返回策略1的全部结果

## API使用

### 端点
```
POST /api/strategy2/screen
```

### 请求参数
```json
{
  "surge_threshold": 1.5,      // 放量阈值（倍）
  "min_change_pct": 3.0,       // 最小涨幅（%）
  "max_stocks": 200,           // 最大筛选股票数
  "hot_sector_only": true      // 是否只返回热点板块股票
}
```

### 响应示例
```json
{

  "strategy": "策略2：策略1+热点板块筛选",
```
